# 日常与健身记录

一款面向健身新手的本地优先训练与状态教练。它把每日状态、饮水、睡眠、疼痛、训练执行和周期复盘放在同一个闭环里，帮助用户判断今天该正常训练、降低强度，还是优先恢复。

## 产品能力

- 今日教练：结合睡眠、饮水、精力、酸痛、疼痛和近期训练给出可解释建议。
- 快速记录：用简短表单完成每日状态、饮水和疼痛记录。
- 新手训练：内置全身、上肢、下肢和恢复模板，支持逐组记录与训练总结。
- 周目标：展示本周训练进度、剩余次数和下一步行动。
- 每周节奏：选择计划训练日，让首页和本地提醒围绕你的固定节奏工作。
- 支持约定与回访：与家人、朋友或教练约定关怀频率、支持方式和沟通边界；每次关怀后记录支持感，帮助调整关系节奏。选择“提醒我完成计划”时，可在邀请中附上计划训练星期。
- 洞察复盘：提供状态评分、趋势、留存回顾、动作进步和周报导出。
- 本地个性化：可设置饮水目标、每周训练目标、训练偏好和保守模式。
- 本地提醒：可设置每日记录与训练提醒；提醒数据不会发送到服务器。
- 数据管理：支持导入前预览、JSON 完整备份、CSV 摘要导出和存储健康检查。
- 安全边界：对疼痛和高风险状态给出保守提示，不提供医疗诊断。
- PWA：支持安装到桌面或手机，并缓存应用外壳供离线打开。

## 快速开始

需要 Node.js 18 或更高版本。

```bash
npm start
```

默认地址：

```text
http://localhost:5173
```

若需要使用云端 AI 建议，可在 PowerShell 中配置：

```powershell
$env:OPENAI_API_KEY="你的 key"
$env:OPENAI_MODEL="gpt-5-mini"
npm start
```

## 生产部署

复制 `.env.example` 中需要的变量到托管平台配置。容器或远程主机通常设置 `HOST=0.0.0.0`；本机运行保留默认 `127.0.0.1`。只有服务位于可信反向代理之后时才设置 `TRUST_PROXY=1`。

托管平台可使用 `GET /api/health` 探活。响应包含服务状态、版本、运行时长和 AI 配置状态，不包含密钥。每个响应均带有 `X-Request-Id`，API 日志可用该值关联一次请求。服务收到 `SIGTERM` 或 `SIGINT` 时会等待当前请求结束后退出。

未配置 `OPENAI_API_KEY` 时，应用会继续使用本地规则生成建议，核心记录功能不受影响。

## 推荐使用流程

1. 在“今天”页查看今日教练建议并完成状态记录。
2. 根据建议开始训练，或选择恢复模板。
3. 在“训练”页逐组记录实际完成情况并保存总结。
4. 在“洞察”页查看周目标、动作进步和近期趋势。
5. 定期在数据设置中导出 JSON 备份；需要表格分析时导出 CSV。

应用内“帮助”页也包含使用流程、安装、提醒、迁移和隐私说明。

公开交付时可直接访问 `/privacy.html` 和 `/terms.html` 查看隐私政策与使用条款；两页也已加入离线应用外壳。

## 安装与离线

在支持 PWA 的浏览器中，可通过页头的安装按钮或浏览器菜单安装应用。首次成功加载后，应用外壳可离线打开；记录仍保存在当前浏览器的本地存储中。

离线能力不等于云同步：更换浏览器、清除站点数据或更换设备前，请先导出 JSON 备份。

## 数据与隐私

- 日常记录、训练记录、动作库、模板、偏好和建议历史默认保存在浏览器 `localStorage`。
- JSON 导出用于完整备份与迁移；导入前会先校验并展示预览，确认后才覆盖当前数据。
- CSV 导出用于表格分析，不替代完整 JSON 备份。
- 浏览器提醒只读取本地记录与设置，不会上传提醒内容。
- 只有在配置 `OPENAI_API_KEY`、用户主动生成并明确同意云端建议时，经过字段白名单和长度限制的近期记录才会发送给 OpenAI API；本地 ID、更新时间、习惯对象和完整动作库不会发送。授权可在帮助页撤回。
- 当前版本没有账号、云同步、订阅或支付系统。

## 验证

语法检查：

```bash
npm run check
```

浏览器冒烟测试：

```bash
npm run test:smoke
```

冒烟测试会启动临时本地服务和无头浏览器，覆盖桌面与手机布局、每日记录、教练建议、训练模板与保存、周目标、洞察、帮助页、导入导出、提醒、安装入口、存储异常和横向溢出。截图保存在 `output/playwright/`。

## 当前产品边界

当前版本聚焦“本地优先的新手训练闭环”，不会提供：

- 医疗诊断或替代专业医疗意见。
- 账号、支付、云同步和多设备协作。
- 公开社区、视频教学或高级周期化训练系统。

这些能力应在核心记录、建议、训练执行、数据可靠性和用户留存得到持续验证后再进入产品范围。
