# 日常与健身记录

一个面向健身新手的本地优先记录工具。它把日常状态、饮水、睡眠、疼痛、训练记录和本地教练建议放在同一个闭环里，帮助用户回答两个问题：

- 今天适合正常练、轻量练，还是恢复？
- 开始训练后，怎么稳稳完成并留下可信记录？

## 当前定位

产品方向是 **健身新手的日常训练与状态教练**，不是泛习惯打卡，也不是高级力量训练表格。

当前差异点：

- 首次引导：用 60 秒记录入口帮助新手先完成最小状态记录。
- 今日教练：根据睡眠、饮水、精力、酸痛、疼痛和最近训练给出建议。
- 新手模板：内置全身、上肢、下肢和恢复训练模板。
- 训练执行：显示计划、进度、完成组数、强度和保存总结。
- 洞察复盘：状态评分、本周回顾、趋势和本地建议。
- 本地优先：基础数据保存在浏览器 localStorage，支持 JSON 导出。

## 快速开始

```bash
npm start
```

默认地址：

```text
http://localhost:5173
```

如需使用云端 AI 建议：

```bash
$env:OPENAI_API_KEY="你的 key"
$env:OPENAI_MODEL="gpt-5-mini"
npm start
```

没有配置 `OPENAI_API_KEY` 时，应用会使用本地规则建议。

## 验证

语法检查：

```bash
npm run check
```

浏览器冒烟测试：

```bash
npm run test:smoke
```

Smoke test 会自动启动临时本地服务和 headless Chrome，检查：

- 今日日期是否使用本地日期。
- 空状态下今日教练是否推荐新手全身训练。
- 今日建议是否能进入训练页并载入模板。
- 模板提示是否不会被误保存为真实训练组。
- 空训练保存是否会被拦截。
- 填写真实训练组后是否能保存并显示总结。
- 桌面和手机宽度是否无横向溢出。

截图输出到：

```text
output/playwright/
```

## 数据与隐私

- 日常记录、训练记录、动作库、模板和建议历史默认保存在浏览器 `localStorage`。
- 导出功能会生成 JSON 备份文件。
- 当前版本没有账号、云同步或订阅系统。
- 如果配置了 `OPENAI_API_KEY`，点击生成建议时会把近期记录发送给 OpenAI API；未配置时只使用本地规则。

## 核心用户路径

1. 打开今天页，查看今日建议。
2. 通过 60 秒记录入口填写睡眠、精力、酸痛和疼痛。
3. 点击开始今天训练，载入新手模板。
4. 在训练页按组记录真实完成情况。
5. 保存训练，查看完成总结。
6. 在洞察页查看状态评分、本周回顾和趋势。

## 商业化路线

当前阶段目标：先把免费核心体验打磨到可信、舒服、可持续。

近期重点：

- 今日教练建议更稳定、更可解释。
- 训练执行体验更像教练陪练，而不是表格。
- 洞察页能把生活状态和训练表现联系起来。
- 数据导出和本地优先体验保持可靠。

后续 Pro 方向：

- AI 周报和训练调整建议。
- 疲劳与疼痛风险提醒。
- 高级趋势图和动作进步分析。
- 云同步、多设备和备份历史。
- 新手进阶训练计划。

## 当前边界

当前版本不会做：

- 医疗诊断。
- 账号和支付。
- 社交分享。
- 视频教学。
- 高级周期化训练系统。

这些能力应在核心记录、建议和执行闭环稳定后再进入。
