# 日常与健身记录

一款面向健身新手的本地优先训练与状态教练。它把每日状态、饮水、睡眠、疼痛、训练执行和周期复盘放在同一个闭环里，帮助用户判断今天该正常训练、降低强度，还是优先恢复。

## 产品能力

- 今日教练：结合睡眠、饮水、精力、酸痛、疼痛和近期训练给出可解释建议。
- 快速记录：用简短表单完成每日状态、饮水和疼痛记录。
- 新手训练：内置全身、上肢、下肢和恢复模板，支持逐组记录与训练总结。
- 周目标：展示本周训练进度、剩余次数和下一步行动。
- 每周节奏：选择计划训练日，记录节奏生效历史并复盘近 4 周的计划兑现率；首页和本地提醒围绕固定节奏工作。
- 个人进展：把最近 28 天的训练、状态覆盖、恢复信号与计划兑现整理为个人阶段报告；达到足够记录后可保守比较此前阶段，并导出不含原始隐私内容的 Markdown 报告。
- 周目标校准：根据最近四个已结束自然周的实际兑现与恢复压力，解释维持或下调建议；只允许用户主动采用，绝不自动加量。
- 支持约定与回访：可分别与多位家人、朋友或教练约定关怀频率、支持方式和沟通边界；每次关怀后的支持感只归属对应伙伴，帮助调整关系节奏。选择“提醒我完成计划”时，只有该伙伴的邀请可附上计划训练星期。
- 训练协作简报：面向教练汇总训练设定、计划节奏、兑现率和执行概览，支持发送前选择披露范围。
- 洞察复盘：提供状态评分、趋势、留存回顾、动作进步和周报导出。
- 本地个性化：可设置饮水目标、每周训练目标、训练偏好和保守模式。
- 本地提醒：可设置每日记录与训练提醒；提醒数据不会发送到服务器。
- 数据管理：支持导入前预览、JSON 完整备份、CSV 摘要导出、Strong/Hevy 训练 CSV 本地迁入和存储健康检查。
- 账号与权益边界：Node 部署可连接 Supabase 邮箱验证码身份和服务器验证的 AI 月度额度；登录不会自动上传本机记录。
- 安全边界：对疼痛和高风险状态给出保守提示，不提供医疗诊断。
- PWA：支持安装到桌面或手机，并缓存应用外壳供离线打开。

## 快速开始

需要 Node.js 18 或更高版本。

```bash
npm start
```

默认地址：

```text
http://localhost:5173
```

## 生产部署

### 使用 Render 部署

项目根目录已经提供 `render.yaml`。将项目上传到 GitHub 后，在 Render 中选择该仓库并创建 Blueprint，Render 会读取这份配置：

1. 创建一个 Web Service，使用项目根目录的 `render.yaml`。
2. 首次公开测试可不配置 `OPENAI_API_KEY`；核心本地记录功能仍可使用。需要启用云端 AI 时，再在服务的 Environment 中单独添加，且不要把真实 key 写入代码或提交到仓库。
3. 等待部署完成，打开 Render 分配的 `https://*.onrender.com` 地址。
4. 访问 `/api/health` 检查服务状态，再把首页地址发给用户。

Render 会提供 HTTPS。生产服务必须使用 `HOST=0.0.0.0`，端口由平台通过 `PORT` 环境变量注入；项目服务已支持这两项配置。

首版数据仍保存在每位用户自己的浏览器中，不会因为部署到 Render 自动变成云端同步。配置 AI 后，用户主动生成建议时，符合页面提示范围的记录会发送到 OpenAI；未配置 key 时，核心本地记录功能仍可使用。

### 可选账号身份

账号不是使用本地记录的前提。需要为未来服务权益建立身份时，可在 Node/Render 部署中同时配置：

```text
SUPABASE_URL=https://你的项目.supabase.co
SUPABASE_ANON_KEY=你的 anon key
```

两项必须同时存在；密钥只配置在服务端环境变量中。应用通过同源服务端代理发送邮箱验证码，认证令牌保存在 `HttpOnly`、`SameSite=Strict` Cookie，页面脚本无法读取。Supabase 邮件模板需要包含 `{{ .Token }}`，让用户收到 6 至 8 位数字验证码。

登录目前只建立稳定用户身份，不会创建训练数据云端副本，也不表示已订阅。GitHub Pages 静态地址没有同源账号服务，因此会诚实显示“本机模式”。

### 可选服务权益与 AI 配额

需要按账号限制云端建议时，先在 Supabase SQL Editor 应用 `supabase/migrations/20260713_entitlements_and_ai_quota.sql`，再在服务端增加：

```text
SUPABASE_SERVICE_ROLE_KEY=你的 service_role key
FREE_AI_ADVICE_LIMIT=3
PRO_AI_ADVICE_LIMIT=100
```

`service_role` 只能存在于服务端环境变量中。配置后，云端建议必须登录，并通过数据库原子预留额度；AI 失败会释放预留。浏览器不能设置 Pro 或修改额度。当前没有购买入口，Pro 只能由未来经过签名验证的支付 webhook 写入；不要通过前端或公开接口手工切换。

复制 `.env.example` 中需要的变量到托管平台配置。容器或远程主机通常设置 `HOST=0.0.0.0`；本机运行保留默认 `127.0.0.1`。只有服务位于可信反向代理之后时才设置 `TRUST_PROXY=1`。

托管平台可使用 `GET /api/health` 探活。响应包含服务状态、版本、运行时长和 AI 配置状态，不包含密钥。每个响应均带有 `X-Request-Id`，API 日志可用该值关联一次请求。服务收到 `SIGTERM` 或 `SIGINT` 时会等待当前请求结束后退出。

未配置 `OPENAI_API_KEY` 时，应用会继续使用本地规则生成建议，核心记录功能不受影响。

## 推荐使用流程

1. 在“今天”页查看今日教练建议并完成状态记录。
2. 根据建议开始训练，或选择恢复模板。
3. 在“训练”页逐组记录实际完成情况并保存总结。
4. 在“洞察”页查看周目标、动作进步和近期趋势。
5. 定期在数据设置中导出 JSON 备份；需要表格分析时导出 CSV；从 Strong 或 Hevy 转入时使用“迁入训练 CSV”。

应用内“帮助”页也包含使用流程、安装、提醒、迁移和隐私说明。

公开交付时可直接访问 `/privacy.html` 和 `/terms.html` 查看隐私政策与使用条款；两页也已加入离线应用外壳。

## 安装与离线

在支持 PWA 的浏览器中，可通过页头的安装按钮或浏览器菜单安装应用。首次成功加载后，应用外壳可离线打开；记录仍保存在当前浏览器的本地存储中。

离线能力不等于云同步：更换浏览器、清除站点数据或更换设备前，请先导出 JSON 备份。

## 数据与隐私

- 日常记录、训练记录、动作库、模板、偏好和建议历史默认保存在浏览器 `localStorage`。
- JSON 导出用于完整备份与迁移；导入前会先校验并展示预览，确认后才覆盖当前数据。
- CSV 导出用于表格分析，不替代完整 JSON 备份。
- Strong/Hevy 训练 CSV 只在当前浏览器解析；确认后去重并追加训练，不覆盖已有记录、设置或动作分类。
- 配置账号服务后，邮箱交由 Supabase Auth 处理；登录会话与本地训练数据分离，邮箱和令牌不进入 JSON/CSV/Markdown 导出。
- 浏览器提醒只读取本地记录与设置，不会上传提醒内容。
- 只有在配置 `OPENAI_API_KEY`、用户主动生成并明确同意云端建议时，经过字段白名单和长度限制的近期记录才会发送给 OpenAI API；本地 ID、更新时间、习惯对象和完整动作库不会发送。授权可在帮助页撤回。
- 当前版本可选提供基础账号身份，但没有云同步、订阅或支付系统。

## 当前产品边界

当前版本聚焦“本地优先的新手训练闭环”，不会提供：

- 医疗诊断或替代专业医疗意见。
- 支付、云同步和多设备协作。
- 公开社区、视频教学或高级周期化训练系统。

这些能力应在核心记录、建议、训练执行、数据可靠性和用户留存得到持续验证后再进入产品范围。
