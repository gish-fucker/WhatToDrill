# 周目标校准设计

## 目标

帮助用户把“每周训练目标”调整到能长期兑现的水平。校准只复用现有训练记录、每日状态和 `weeklyWorkoutTarget`，不增加第二套计划模型，不制造失败感，也不以完成更多训练为理由自动提高目标。

该能力属于免费个人复盘闭环的一部分。它通过更可信、更可持续的建议提升留存和产品信任，为未来的长期报告、阶段目标和云端深度复盘建立付费价值基础，但本身不锁入 Pro。

## 方案选择

采用“28 天个人进展报告内建议并可直接采用”的方案。

未采用的方案：

- 只在偏好设置显示：修改路径清晰，但用户很难在有充分上下文时发现建议。
- 报告与设置同时显示：发现性更强，但首版会重复解释同一结论并增加状态同步成本。

报告是用户理解最近节奏和恢复信号的地方，因此校准结论与行动应在同一个上下文内完成。偏好设置继续保留手动编辑入口。

## 数据范围

只读取最近四个已经结束的本地自然周，周一至周日。当前尚未结束的自然周不参与校准，避免把仍有机会完成的目标判为失败。

每周统计：

- 训练记录次数。
- 是否达到当前 `weeklyWorkoutTarget`。
- 是否有每日状态或训练记录，用于判断该周是否被观察到。

恢复信号读取同一四周内的每日记录：睡眠、精力、酸痛和疼痛。备注、动作名称、重量、次数、记录 ID 和具体训练日期不进入结论文本。

## 数据门槛

只有同时满足以下条件才给出校准结论：

- 最近四个完整周中至少三个周有每日状态或训练记录。
- 四周内至少有 6 条每日状态记录或 3 次训练记录。

不满足时显示“继续建立校准依据”，说明还需要覆盖至少三个完整周。此时不显示采用按钮，不把空白周当作漏练。

## 校准规则

校准结果只有“维持”和“建议下调”两类，不产生上调建议。

### 建议维持

满足任一情况时维持当前目标：

- 当前目标已经是每周 1 次。
- 最近被观察到的完整周中，至少两个周达到当前目标。
- 虽未稳定达标，但证据不足以证明当前目标不可持续。

当用户稳定完成超过目标的训练次数时，文案肯定其稳定性，但仍维持当前目标，不诱导加量。

### 建议下调

当前目标大于 1，且至少三个完整周被观察到时，仅在以下情况建议下调 1 次：

- 达标周不超过一个，并且被观察周的训练次数中位数低于当前目标；或
- 至少两个被观察周未达标，同时存在明确恢复压力：平均睡眠低于 6.5 小时、平均疼痛不低于 2/5，或出现至少两天疼痛 4/5 以上。

推荐值始终为 `max(1, 当前目标 - 1)`。一次校准最多下降 1，不根据单次高疼痛或单个漏练周直接修改目标。

疼痛相关文案只说明“优先恢复并考虑较低目标”，同时保留现有的非医疗诊断边界；不把校准描述为治疗建议。

## 界面与交互

在 28 天个人进展报告的发现列表之后增加“周目标校准”区域，包含：

- 当前目标与建议目标。
- 一句主要结论。
- 一句可解释原因，说明基于完整周的兑现情况或恢复压力。
- 数据不足时的下一步记录提示。
- 仅在建议值低于当前值时显示“采用每周 N 次”按钮。

点击采用后：

1. 将现有 `weeklyWorkoutTarget` 更新为建议值。
2. 通过现有设置标准化与本地保存流程持久化。
3. 重新渲染周目标、个人进展报告和偏好设置。
4. 显示明确成功提示。

不弹出二次确认。该操作可随时在偏好设置中手动改回，不删除或改写任何历史记录。

## 导出与隐私

个人进展 Markdown 可增加“周目标校准”汇总结论，包括当前目标、建议目标和原因类别。导出不得包含：

- 每周或每天的具体训练日期。
- 每周训练次数明细数组。
- 单日睡眠、疼痛、精力或酸痛值。
- 备注、动作名称、重量、次数和记录 ID。

应用不自动上传校准数据，不新增持久化健康字段。

## 错误与边界

- 非法或缺失的周目标继续由现有 `sanitizeWeeklyWorkoutTarget` 处理。
- 相同日期的多次训练按真实训练记录次数计入，与现有周目标口径一致。
- 跨月、跨年仍按本地自然周计算。
- 数据中位数只基于被观察周，避免把未使用应用的空白周视为失败。
- 应用刷新后不保存“推荐结果”；每次都从当前本地数据重新计算。
- 采用建议后立即重新计算，结果应变为维持新目标或等待更多证据，不重复显示同一个下调按钮。

## 验收标准

- 数据不足时不产生维持或下调结论，也不显示操作按钮。
- 当前周未完成不会影响校准。
- 稳定达标时维持当前目标；超额完成时不建议上调。
- 多周无法兑现时最多建议下调 1 次，且不低于每周 1 次。
- 恢复压力只在有足够完整周证据时参与下调建议。
- 点击采用后，本地设置、周目标面板、报告和偏好设置同步更新。
- 导出文本只含汇总结论，不泄露具体日期或原始健康与训练内容。
- 桌面和 390px 移动视口无横向溢出，按钮文本完整可见。
- 现有语法检查和完整浏览器冒烟测试继续通过。
