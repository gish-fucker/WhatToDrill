# 每周训练节奏设计

## 目标

把“每周训练次数”从抽象目标转化为用户可坚持的重复节奏。用户选择计划训练日后，首页显示下一次安排，本地训练提醒只在计划日触发，从而减少无关打扰并提升按计划完成的概率。

## 方案取舍

首版采用本地重复周计划，不做账号日历、预约链接或多端同步。固定星期足够覆盖多数入门用户，也能为未来 Pro 周期计划、教练排期和家庭协作提供稳定的数据结构，同时不把个人行程交给服务端。

## 交互与规则

数据设置中加入周一至周日的多选项，允许留空。留空时保留现有“周目标未完成”提醒逻辑；选择后，训练提醒仅在计划日且当天尚未完成训练时触发。周目标面板增加“计划节奏”，展示选中的星期以及下一次计划日；今天是计划日时明确显示。

## 数据与隐私

`settings.plannedWorkoutDays` 是去重、排序后的星期索引数组（0 为周日，1 为周一至 6 为周六）。导入时只保留 0 至 6 的整数，随 JSON 备份保存。计划日只描述训练意向，不包含日程标题、地点、联系人、健康或训练原始数据，且不会发送到云端或支持邀请。

## 验收

- 用户可保存、重载和导入合法计划日；非法值会被过滤。
- 周目标面板显示计划日与下一次安排。
- 启用训练提醒后，计划存在时仅在计划日触发；未设计划时保持原行为。
- JSON 完整备份保留计划日，隐私政策说明本地保存范围。
- 桌面和 390px 移动端无横向溢出，完整冒烟测试通过。
