# P1 品牌与导航收口设计

日期：2026-07-15  
状态：用户已确认，直接实施  
范围：P1 的品牌层级、一级导航和术语统一；不引入新的训练、付费、同步或洞察能力。

## 1. 背景

P0 已将首次使用路径收敛为“今天该怎么练 → 开始训练”。但当前公开页和应用仍把“今天练什么”当作产品名称，并保留“洞察、动作与模板、帮助”等五个并列导航入口。用户会在产品名称、日常状态和健身训练之间看到多个平行概念，难以形成“训练与恢复助手”的整体理解。

本轮不更换正式产品名。"下一练"等更聚焦的候选名保留给未来独立品牌决策；本次先让现有名称与用户价值表达一致。

## 2. 目标与非目标

### 2.1 目标

1. 全站以“日常与健身记录”为品牌名，以“你的训练与恢复助手”为固定价值描述。
2. “今天练什么”只作为今天页的任务标题或行动文案，不再承担全站品牌名。
3. 消除“日常记录”和“健身记录”两个并列产品概念：前者统一表述为“恢复状态”，后者统一表述为“训练”。
4. 一级导航从五项收敛为“今天 / 训练 / 进步 / 我的”。
5. 既有记录、模板、导入导出、提醒、支持伙伴、帮助和设置仍可访问，不删除任何数据或功能。

### 2.2 非目标

- 不把产品正式改名为“下一练”“训练节奏”或其他候选名。
- 不在本轮新增继续上次训练、渐进加重、7—14 天个人规律、云同步、支付或 Pro 权益。
- 不重写训练会话、恢复状态判定或数据模型。

## 3. 品牌层级

### 3.1 全站表达

| 位置 | 文案 | 作用 |
| --- | --- | --- |
| 浏览器标题、落地页和应用页品牌位 | 日常与健身记录 | 稳定产品名 |
| 品牌副标题 / 描述 | 你的训练与恢复助手 | 用一句话解释价值 |
| 今天页核心任务 | 今天练什么？ | 将用户带回当日训练决策 |

落地页仍以“今天练什么，直接告诉你”表达即时收益，但该句是主张，不是产品名。页面品牌位必须先显示“日常与健身记录”，并在可见范围内说明“你的训练与恢复助手”。

### 3.2 术语规则

| 旧概念 | 新概念 | 使用边界 |
| --- | --- | --- |
| 日常记录 | 恢复状态 | 睡眠、精力、酸痛、疼痛及其补充信息；服务于训练决策 |
| 健身记录 | 训练 | 训练计划、逐组完成、训练历史和模板 |
| 洞察 | 进步 | 动作对比、趋势、阶段回顾与下一步建议 |

历史导出标题和数据键不因本轮更改，避免破坏兼容性；仅更新用户可见的名称和说明。

## 4. 一级导航与归属

| 一级入口 | 用户问题 | 保留内容 | 收纳内容 |
| --- | --- | --- | --- |
| 今天 | 我今天该怎么练？ | 今日建议、恢复状态快速调整、疼痛提示、开始训练 | 不展示完整数据管理或支持伙伴 |
| 训练 | 我正在练什么，练过什么？ | 聚焦训练会话、训练记录、训练模板入口 | 动作库与模板管理可进入“我的”管理页 |
| 进步 | 我有没有在进步，下一步怎么做？ | 动作进步、趋势、周回顾、阶段报告 | 不承担设置或导出入口 |
| 我的 | 我如何管理自己的训练工具？ | 动作库、模板管理、偏好、提醒、数据、支持伙伴、帮助、账号与安装状态 | 不再额外占用一级导航 |

“恢复状态”不是一级导航。它是“今天”页中帮助用户决定今天训练强度的次入口，避免再次形成与训练平行的一套记录产品。

## 5. 交互与兼容性

1. 原有顶栏五个标签改为四个，活动状态、键盘导航、ARIA 标签与移动端宽度同步更新。
2. 原“动作与模板”和“帮助”的内容迁入“我的”页内的清晰二级入口；现有按钮与数据不失效。
3. 用户通过旧的程序内事件或恢复状态进入相关功能时，应落到对应的新入口，不产生空白页。
4. 训练中仍保持 P0 的当前组聚焦；品牌和导航调整不能让训练过程中出现额外干扰。
5. 浏览器标题、HTML 描述、PWA 名称与公开落地页采用同一品牌层级；PWA 安装入口仍直接启动 `/app/`。

## 6. 验收标准

- 首次访问落地页，可见“日常与健身记录”和“你的训练与恢复助手”，并能在 5 秒内理解产品做什么。
- 应用页不再把“今天练什么”作为全站品牌名。
- 一级导航只显示“今天 / 训练 / 进步 / 我的”，桌面与 390px 手机宽度均无横向溢出。
- 用户可从“我的”访问动作与模板、帮助、提醒、数据与设置；既有功能可用。
- 用户可在“今天”调整恢复状态并开始训练；训练、历史和洞察数据不丢失。
- `npm.cmd run check`、`npm.cmd run test:workout-session`、`npm.cmd run test:smoke` 均通过，并用真实浏览器验证落地页、四栏导航和移动端布局。

## 7. 实施顺序

1. 更新公开落地页、HTML 元数据、PWA 元数据与应用品牌位。
2. 将五栏顶级导航改为四栏，并建立“我的”页的二级入口结构。
3. 迁移动作与模板、帮助、提醒、数据和支持伙伴的入口，不改其底层数据逻辑。
4. 统一用户可见的“日常记录 / 健身记录 / 洞察”术语。
5. 更新冒烟测试断言，执行桌面与手机浏览器回归。

## 8. 自检结论

本设计没有未定文案、占位项或与 P0 路径相冲突的要求。它只收敛信息架构和命名：今天页仍优先开始训练，恢复状态仍是可选调整，训练会话和正式历史的数据模型不变。范围保持在 P1 的品牌与导航收口内。
