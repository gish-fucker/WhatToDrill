# 小习惯与健身记录软件第一版设计

## 目标

第一版做成本地网页原型。用户打开本地服务后，可以快速记录日常生活状态、健身训练内容，并在同一界面查看近期趋势和建议。

## 范围

第一版包含：

- 日常记录：睡眠、饮水、心情、精力、酸痛、疼痛、常用习惯打卡、自由备注。
- 健身记录：训练主题、训练时长、整体难度、动作、组数、次数、重量、单组难度、备注。
- 动作库：维护常用动作，记录分类和最近使用。
- 训练模板：把一组常用动作保存为模板，并在之后快速载入。
- 历史与趋势：查看最近记录、训练总量、生活状态变化。
- AI 建议：有 `OPENAI_API_KEY` 时调用 OpenAI Responses API；没有 key 时使用本地规则给出可用建议。
- 数据导入导出：通过 JSON 备份或迁移本地数据。

第一版不包含：

- 用户登录、多设备同步、云数据库。
- 医疗诊断、伤病治疗方案。
- 复杂周期计划、营养精算、动作视频分析。

## 产品结构

应用分为四个主页面：

1. 今天：填写当天生活状态和习惯。
2. 训练：记录本次训练，支持添加动作和组，支持从模板载入。
3. 洞察：展示最近趋势、训练历史、智能建议。
4. 动作与模板：管理动作库、模板、数据导入导出。

首页默认进入今天记录，适合每天打开就用。

## 数据模型

数据保存在浏览器 `localStorage` 中，主键为 `habit_fitness_app_v1`。

核心对象：

- `dailyLogs`：每日生活状态。
- `workouts`：训练记录。
- `exercises`：动作库。
- `templates`：训练模板。
- `adviceHistory`：建议历史。

日常记录按日期覆盖保存，同一天多次保存会更新当天记录。训练记录按每次训练新增保存。

## AI 建议流程

用户在洞察页点击生成建议：

1. 前端整理最近 14 天日常记录、最近 10 次训练、动作趋势。
2. 如果本地服务配置了 `OPENAI_API_KEY`，请求 `/api/advice`，由模型生成建议。
3. 如果没有配置 key 或请求失败，前端使用本地规则生成建议。
4. 建议保存到 `adviceHistory`，便于之后回看。

AI 输出聚焦：

- 最近状态总结。
- 训练进步或停滞。
- 恢复、睡眠、疼痛风险提醒。
- 下一次训练建议。
- 一个最值得优先改进的小习惯。

## 技术方案

- 前端：原生 HTML、CSS、JavaScript，无构建步骤。
- 本地服务：Node.js 内置 `http` 模块，无 npm 依赖。
- 数据持久化：浏览器 `localStorage`。
- AI：服务端读取环境变量 `OPENAI_API_KEY`，使用 OpenAI Responses API。

这种方案能在当前空目录里快速产出可用版本，同时保留后续迁移到 React、SQLite、桌面端或移动端的空间。

## 验证标准

第一版可用的判断：

- 能打开本地网页。
- 能保存并刷新后保留日常记录。
- 能保存训练记录，包括多个动作和多组数据。
- 能新增动作，保存模板，并从模板载入训练。
- 能查看最近趋势和历史。
- 能生成智能建议；没有 API key 时也能输出本地建议。
- 能导出和导入 JSON 数据。
