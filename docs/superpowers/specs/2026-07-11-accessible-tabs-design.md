# 可访问主导航设计

## 目标

让主导航符合标准标签页交互，使键盘和读屏用户能识别当前页面、快速切换并跳过重复导航。

## 语义

- 主导航使用 `role=tablist`，五个按钮使用 `role=tab`。
- 每个标签通过 `aria-controls` 关联唯一的 `tabpanel`，面板通过 `aria-labelledby` 反向关联。
- 当前标签使用 `aria-selected=true` 和 `tabindex=0`；其他标签为 `aria-selected=false` 和 `tabindex=-1`。
- 非当前面板设置 `hidden`，避免读屏进入不可见内容。

## 键盘

- `ArrowRight` / `ArrowLeft` 循环切换相邻标签。
- `Home` / `End` 跳到第一个或最后一个标签。
- 键盘切换同时激活面板并移动标签焦点，但不触发页面滚动。
- 鼠标、触屏及应用内部切页继续沿用现有滚动行为。

## 跳转与焦点

- 应用和两份政策页首项都提供“跳到主要内容”链接，仅在获得焦点时显示。
- 主内容可被程序聚焦，但不进入普通 Tab 顺序。
- 为通用交互控件提供清晰的 `focus-visible` 外框。

## 验证

- 自动化测试检查角色、关联、选中状态、停靠顺序和隐藏面板。
- 模拟方向键、Home、End，确认激活面板和焦点同步。
- 检查跳转链接目标及桌面、手机无横向溢出。
