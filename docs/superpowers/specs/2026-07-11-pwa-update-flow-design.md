# PWA 更新流程设计

## 目标

让已安装或长期打开应用的用户明确知道新版本可用，并由用户选择更新时间，避免后台切换后界面仍停留在旧代码。

## 交互

- 新工作线程安装完成后，在应用页头下方显示持久更新横幅。
- 横幅提供“立即更新”和“稍后”两个操作；稍后只关闭本次提示，下次载入仍可检测等待中的版本。
- 点击立即更新后，通知等待中的工作线程接管；控制器变化后页面只刷新一次。
- 不自动刷新，避免打断正在填写的日常记录。训练草稿继续使用现有自动保存机制。

## 缓存生命周期

- 首次安装按浏览器标准流程激活。
- 后续版本不在 `install` 阶段自动调用 `skipWaiting`，而是等待用户确认。
- 工作线程响应 `SKIP_WAITING` 消息后激活，并在激活时清理旧缓存、接管页面。

## 版本展示与验证

- 应用版本集中定义为 `1.1.0`，帮助页显示同一版本。
- 自动化测试验证等待版本提示、稍后关闭、立即更新消息以及版本号。
- 现有 PWA 缓存、桌面和手机核心流程继续完整回归。
