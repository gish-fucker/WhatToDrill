# 训练 CSV 迁移设计

## 目标

让已有 Strong 或 Hevy 训练历史的用户在当前设备上安全迁入数据，降低更换产品的成本。迁移强调数据可携带性和用户控制：文件只在浏览器解析，先预览、后确认，合并而不是覆盖现有记录。

该能力服务于商业化获客和信任建设。它不作为 Pro 收费项，也不要求账号、云同步或上传文件。

## 方案选择

采用“Strong/Hevy 格式预设识别 + 本地合并预览”。

未采用的方案：

- 只重新导入本应用 CSV：现有摘要 CSV 不包含完整动作组，无法恢复训练细节，也不能解决竞品迁移。
- 通用字段映射向导：兼容面更广，但首版交互和错误空间过大，用户难以判断单位、训练分组和日期格式。

首版只对有公开样例证据的 Strong 与 Hevy 列结构声明支持。未知格式明确拒绝，不猜测字段。

## 支持格式

### Strong

必需列：

- `Date`
- `Workout Name`
- `Exercise Name`
- `Set Order`

可选读取：`Duration`、`Weight`、`Reps`、`Distance`、`Seconds`、`Notes`、`Workout Notes`、`RPE`。

### Hevy

必需列：

- `title`
- `start_time`
- `exercise_title`
- `set_index`

可选读取：`end_time`、`description`、`set_type`、`weight_kg`、`reps`、`distance_km`、`distance_meters`、`duration_seconds`、`rpe`、`exercise_notes`、`workout_duration`。

列名去除 UTF-8 BOM 和首尾空白后进行精确匹配，大小写按各来源格式处理。缺少必需列时阻止导入并列出缺失项。

## CSV 解析

浏览器内使用状态机解析逗号、双引号、双引号转义、CRLF/LF 和带换行的引号字段。不得用简单 `split(",")`。

限制：

- 文件最大 5 MB。
- 最多 20,000 条数据行。
- 空行忽略。
- 列数超过表头的行视为无效。
- 解析到未闭合引号时阻止导入。

## 标准化

### 训练分组

- Strong 使用原始 `Date + Workout Name` 分组。
- Hevy 使用原始 `start_time + title` 分组。
- 同一天同名但开始时间不同的训练保持为两次记录。

### 日期与时长

- 支持 `YYYY-MM-DD`、`YYYY-MM-DD HH:mm:ss` 和 Hevy 常见的 `DD Mon YYYY, HH:mm`。
- 应用只保存本地日期 `YYYY-MM-DD`，不保存原始时区或精确开始时间。
- Strong 时长支持 `35m`、`1h 43m`；Hevy 优先用开始/结束时间差，其次读取 `workout_duration`。
- 时长限制在 0 至 600 分钟；无效值记为空，不阻止整次训练。

### 动作与组

- 每次训练按动作名称合并连续或分散行，并按来源组序排序。
- 重量、次数和 RPE 只接受有限数字；重量和次数不得为负，RPE 限制在 1 至 10。
- 距离、秒数、非普通组类型和动作备注合并到该组备注，避免静默丢失本应用暂无专用字段的数据。
- 训练备注读取 Strong `Workout Notes` 或 Hevy `description`。
- 训练整体 RPE 使用有效组 RPE 的平均值；没有组 RPE 时使用保守默认值 6。

### 动作库

迁移成功后，把新动作名称合并到现有动作库，默认分类为“其他”，不删除或改写已有分类。

## 去重与合并

导入训练通过以下规范化摘要判断重复：

- 本地日期。
- 去空白并转小写的训练标题。
- 动作名称顺序。
- 每组重量、次数和 RPE。

与现有训练摘要相同的记录计为重复并跳过。当前文件内部的重复训练同样只保留一次。

确认迁移时：

- 只追加非重复训练。
- 不覆盖日常记录、模板、偏好、建议历史或已有训练。
- 不修改现有记录 ID。
- 新训练生成本地 ID 和创建时间。

## 预览与交互

数据设置区域增加“迁入训练 CSV”文件按钮，与 JSON 完整备份导入分开。

预览展示：

- 识别来源。
- 可新增训练次数。
- 动作数和组数。
- 重复训练数。
- 跳过行数和警告。

确认按钮使用“合并 N 次训练”，明确不会覆盖。未知格式、无有效训练、全部重复或解析错误时禁用确认按钮。用户可取消并重新选择文件。

JSON 导入继续保留“覆盖完整本地数据”的现有语义和独立文案。

## 隐私与安全

- CSV 内容不上传服务器，不进入云端建议请求。
- 预览只在当前页面内存保存，取消或刷新后丢弃。
- 导入后内容按现有本地存储、导出和删除规则处理。
- 文件名、原始开始时间和来源应用不持久化到训练记录。
- 训练与健康建议仍不构成医疗诊断。

## 验收标准

- Strong 与 Hevy 公开样例列结构能被正确识别。
- 引号、逗号、换行和 BOM 可正确解析；未闭合引号被阻止。
- 多行组记录正确聚合为训练、动作和组。
- 日期、时长、重量、次数、RPE、备注和特殊组信息按规则保留。
- 无效行可跳过并在预览说明，关键列缺失或无有效训练时阻止导入。
- 重复训练不会追加；确认迁移不覆盖任何现有数据。
- 新动作进入动作库，已有动作分类保持不变。
- CSV 文件在确认前不会修改 `state` 或本地存储。
- 桌面和 390px 手机预览、指标和操作按钮无横向溢出。
- 隐私政策、帮助和 README 说明本地迁移范围。
- 现有 JSON 备份导入和完整冒烟测试继续通过。
