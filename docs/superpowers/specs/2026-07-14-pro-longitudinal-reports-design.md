# Pro 纵向进展报告设计

## 目标

在免费 28 天个人进展报告之上，为服务器验证的 Pro 用户提供 90 天和年度纵向报告。报告帮助用户回答：长期训练是否持续、恢复信号是否稳定、哪些阶段变化值得保留，以及下一阶段怎样安排更可持续。

基础记录、安全提示、周复盘、28 天报告、备份和导出继续免费。功能不依赖或扩展“支持伙伴”。

## 方案选择

采用“服务器验证权益，本机生成确定性报告”。浏览器通过现有 `/api/account/entitlements` 获取服务器验证的 Pro 状态；报告计算只读取当前设备的本地记录，不上传长期健康或训练历史。

未采用：

- 浏览器本地保存 Pro：可任意篡改，与现有权益边界冲突。
- 服务端生成完整报告：需要上传长期健康与训练数据，当前隐私、删除和同步能力尚不足以承担该范围。
- 把 90 天报告全部免费：会削弱首个付费层的具体价值，但 28 天免费报告已足以让用户先获得真实收益。

本地 JavaScript 不是防盗版安全边界。界面只接受服务器权益结果，但真正消耗云端成本、使用私有模型或需要严格授权的能力仍必须由服务端保护。

## 可见状态

新增“长期进展”区域，位于免费 28 天报告之后。

- 静态部署或未配置权益服务：完全隐藏，不展示虚假购买入口。
- 已配置权益但未登录：显示能力说明和“前往账号”命令，不显示报告数据。
- 已登录 Free：显示 90 天与年度报告包含的内容、当前数据准备度和“付费方案尚未开放”，不显示购买按钮。
- 权益服务失败：显示“权益状态暂不可用”，不猜测 Free 或 Pro。
- 已验证 Pro：显示 90 天/年度分段选择、准备度、报告指标、阶段发现和本地导出命令。

权益变化只保存在运行时变量，不进入业务 `state` 或 `localStorage`。

## 数据门槛

### 90 天报告

- 窗口：今天向前 90 天。
- 最小数据量：至少 18 天状态记录或 8 次训练。
- 阶段比较：窗口前 30 天与后 30 天都至少有 6 天状态或 3 次训练时才生成比较结论。

### 年度报告

- 窗口：今天向前 365 天。
- 最小数据量：记录跨度至少 180 天，并且至少有 60 天状态记录或 24 次训练。
- 阶段比较：最早 90 天与最近 90 天都满足 90 天门槛时才生成比较结论。

不足时显示明确缺口，不生成趋势或身体变化结论。导入记录与手工记录使用同一规则。

## 报告内容

报告指标仅使用派生汇总：

- 训练次数、完成组数、状态覆盖天数。
- 有训练的自然周数、有记录的自然月数。
- 最长连续有训练周数。
- 平均睡眠、平均疼痛和高疼痛记录次数。
- 首尾阶段的训练频率与恢复趋势，仅在两段数据都达到门槛时显示。

报告发现使用中性、可解释文案：节奏更稳定、节奏放慢、恢复较稳定、恢复压力需要优先、数据仍不足。不得推断增肌、减脂、体型、疾病、训练效果或因果关系。

行动建议最多三条，优先级依次为安全恢复、保持可持续频率、提高记录覆盖。不会自动修改周目标。

## 组件边界

- `buildLongitudinalWindow(start, end)`：生成不含原始记录的窗口汇总。
- `buildProLongitudinalReport(period)`：应用门槛、阶段比较和行动规则。
- `buildProLongitudinalReportText(report)`：生成可导出的 Markdown。
- `renderProLongitudinalReport()`：只处理权益状态、周期选择和 DOM。
- `exportProLongitudinalReport()`：本地下载，失败时复制文本。

周期选择只在当前页面运行时保存，不新增持久化字段。

## 隐私与安全

报告不读取或输出备注原文、动作名、重量、次数、单次 RPE、具体训练日期、本地 ID 或完整历史。导出文件标明在当前设备生成、不构成医疗诊断或身体成分评估。

登录、权益读取、周期切换和报告导出不得修改本地业务记录。退出或权益降级后立即隐藏 Pro 报告内容，但不删除任何本地记录。

## 测试

- 静态/未配置、未登录、Free、Pro、权益失败五种界面状态。
- 90 天和年度数据门槛、跨度门槛及缺口文案。
- 首尾阶段数据不足时不生成比较；足够时只生成与真实输入一致的趋势。
- 活跃周、活跃月和最长连续训练周跨年仍正确。
- 导出不包含备注、动作、重量、次数、具体日期或 ID。
- 权益变化和导出不改变业务 `localStorage`。
- 桌面与 390px 手机无横向溢出，分段控件尺寸稳定。
- 既有 28 天报告、周目标校准、账号 UI、云端配额和离线能力继续通过完整冒烟测试。

## 发布边界

本阶段不接入支付、不提供升级按钮、不手工写入 Pro，也不承诺云同步。Free 用户看到的是可兑现的能力说明和数据准备度；待真实商户信息、价格和退款政策齐备后，支付 webhook 才能更新同一服务端权益记录。
